# /decision — Full Decision Support

**Purpose**: Structure a decision with options, evaluation matrix, and a clear recommendation.

**When to use**: Specific binary or multi-option decision with meaningful stakes.

---

## Phase 1: Frame (if not already done)

Confirm: decision statement, decision-maker, timeline, reversibility.

Key question: **Is this reversible?** Reversible decisions should be made fast and at low level. Irreversible decisions warrant the full /decision treatment.

---

## Phase 2: Options

List the genuine options. Usually 2–4. Avoid false choices.
- Include the "do nothing" option if it's genuinely available
- Include any non-obvious options that should be on the table
- Combine similar options into one
- Kill any option that can't be executed with the resources available

---

## Phase 3: Evaluation criteria

Name 4–6 criteria that matter for this specific decision. Don't use generic criteria — make them specific to the situation.

Assign weights (totalling 100%) based on what actually matters.

Common criteria families:
- Strategic impact (fit, differentiation, moat value)
- Financial return (NPV, payback, upside magnitude)
- Execution risk (complexity, capability, timeline)
- Capital requirement (amount, reversibility)
- Competitive dynamics (how competitors respond)
- Optionality (what doors does this open or close?)

---

## Phase 4: Evaluation matrix

Score each option against each criterion (1–5 scale). Multiply by weight. Sum.

| Criterion | Weight | Option A | Option B | Option C |
|---|---|---|---|---|
| [criterion 1] | 30% | 4 | 3 | 2 |
| [criterion 2] | 25% | 2 | 4 | 3 |
| ... | ... | ... | ... | ... |
| **Weighted total** | 100% | **X.X** | **X.X** | **X.X** |

Scores are inputs, not outputs. If the matrix says one thing and judgment says another, interrogate the criteria weights, not the judgment.

---

## Phase 5: Recommendation

Lead with the recommendation. Then rationale. Then key risks. Then ask.

**Do not**: present the matrix and invite the reader to decide. The consultant's job is to have an opinion.

If you genuinely can't recommend one option, name what additional information would break the tie — and get it.

---

## Output format

```
RECOMMENDATION: [Option X]

RATIONALE: [2–3 sentences: why this option, what it gets us, what it costs]

KEY RISKS:
  - [risk 1] | Mitigation: [action]
  - [risk 2] | Mitigation: [action]

WHAT THIS REQUIRES: [resources, decisions, sequencing]

NEXT STEP: [single specific action to start executing]

DECISION MATRIX: [table — include if decision was close or stakeholders will want to see the logic]
```
